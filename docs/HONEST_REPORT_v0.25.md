## 2. Архитектура детекции (detect_ai)

Комбинированная формула:  
```
score = heuristic × 0.05 + stat × 0.15 + neural × 0.80
```

- **Heuristic** (5%) — паттерны AI-текста (стартовые слова, буквальные фразы)
- **Statistical** (15%) — `AIDetector`: n-граммы, энтропия, burstiness
- **Neural MLP** (80%) — 35→64→32→1 с ReLU, z-score нормализация

Классификация: >0.55 = "ai", >0.34 = "mixed", ≤0.34 = "human"

**Вывод**: Наш гуманизатор эффективно убивает статистический детектор (0.95→0.05), но **нейронный MLP остаётся на уровне 0.68-0.72**. Нейронный детектор составляет 80% итогового скора, поэтому bypass невозможен.

## 4. Корневая причина

Нейронный MLP анализирует 35 текстовых признаков. Ключевые из них, которые наш гуманизатор НЕ может эффективно изменить:

| Признак | AI-текст | После гуманизации | Человек | Вес |
|---------|----------|-------------------|---------|-----|
| word_length_variance | 13.5 (z=+3.0 max) | 6.5 (z=+2.5) | 5.0 (z=+1.2) | +0.37 |
| avg_syllables_per_word | 2.75 (z=+3.0 max) | 1.71 (z=+0.97) | 1.30 (z=-0.91) | +0.20 |
| sentence_length_variance | 4.7 (z=-1.0) | 232 (z=+3.0 max!) | 38 (z=+0.33) | +0.91 |
| burstiness_score | -0.81 (z=-3.0 max) | -0.20 (z=0.0) | -0.34 (z=-0.75) | +1.15 |
| starter_diversity | 1.0 (z=+1.67) | 1.0 (z=+1.67) | 0.86 (z=+0.87) | — |

**Проблема**: Чтобы реально обмануть нейронный детектор, нужно:
1. Менять длину слов (заменять длинные на короткие) — рискует качеством текста
2. Менять структуру предложений (вариативность длин) — рискует связностью
3. Добавлять "человеческую хаотичность" — рискует читабельностью

Мы реализовали `_normalize_word_length_variance()` и `_normalize_sentence_lengths()`, но **агрессивные замены разрушают текст** (пример: "language" → "albanian" из тезауруса Moby без учёта контекста). Мы исправили это, но now замены консервативные и эффект слабее.

### Почему мы проигрываем

**Фундаментальная причина**: конкуренты используют LLM (GPT-4, Claude, etc.) для полного перефразирования текста. Это создаёт НОВЫЙ текст с теми же идеями, но абсолютно другой структурой. Нейронные детекторы обучены на статистических паттернах AI-генерации — LLM-перефразирование создаёт текст с human-like паттернами потому что LLM может контролировать:
- Длину слов и предложений
- Лексическое разнообразие  
- Структуру аргументации
- "Человеческие" ошибки и неформальности

**Наш подход** (rule-based трансформации):
- Заменяет отдельные слова → не меняет общую статистику длин
- Инжектирует дискурсивные маркеры → ломает структуру предложений
- Разбивает предложения → создаёт неестественные фрагменты
- Не понимает контекст → заменяет слова на семантически неверные

## 6. Что было сделано в этой сессии

### Исправления качества
1. **Убрана SynonymDB-обогащение в `_replace_vocab()`** — SynonymDB добавляла тезаурусные записи к кураторским замена, что приводило к "particularly" → "alonely"
2. **Blacklist этнонимов в SynonymDB** — Moby Thesaurus содержит гипонимы ("language" → "albanian", "afrikaans", "algonquin"). Добавлено ~200 этнонимов/языковых имён в чёрный список
3. **Исправлена морфологическая система** — добавлены guards для суффиксов "-er" ("moreover" → "and" вместо "ander") и "-ly" ("significantly" → "a lot" вместо "muchly")  
4. **Удалена замена "arguably"** — это слово специально инжектируется пайплайном, замещать его контрпродуктивно

### Новые методы
5. **`_normalize_word_length_variance()`** — итеративная замена длинных слов короткими (только из проверенных словарей)
6. **`_normalize_sentence_lengths()`** — разбивка предложений >28 слов
7. **Расширен `_EXTRA_LONG_SHORT`** — ~150 EN, ~25 RU, ~14 UK проверенных замен

### Метрики
- **2045/2045 тестов проходят** ✅
- Средний detect_ai скор: 0.867 → 0.582 (Full pipeline) = **-33%**
- Bypass rate: 0/5 = **0%** (порог <0.40)

## 7. Предложения по улучшению

### Краткосрочные (rule-based, без LLM)

1. **Улучшить sentence restructurer** — текущий ломает структуру предложений. Нужна переработка с учётом грамматических связей (подлежащее-сказуемое не разбивать)

2. **Neural-aware feedback loop** — после трансформаций вычислять нейронные признаки и итеративно подгонять текст, пока score не упадёт ниже порога

3. **Контекстная замена слов** — перед заменой проверять N-грамму окружения. Если bigram "natural|language" или "machine|learning" — не заменять

4. **Starter diversity** — текущий `_normalize_starter_diversity()` не эффективен (diversity=1.0 вместо цели 0.7)

### Среднесрочные (LLM-гибрид)

5. **Интеграция с локальными LLM** (уже частично реализовано через `backend="ollama"`):
   - Ollama с Llama 3.2 для перефразирования ключевых предложений
   - Только предложения с высоким neural score → экономия API
   - Это единственный путь к реальному bypass

6. **Обучение собственной модели перефразирования**:
   - Fine-tune Llama 3.2 (или Gemma 2B) на парах "AI text → human text"
   - Маленькая модель (2-3GB) может работать локально
   - Даст 80%+ bypass без облачного API

### Долгосрочные

7. **Адаптивная стратегия под конкретный детектор**:
   - GPTZero, Originality.ai, Copyleaks используют разные модели
   - Нужен профиль bypass-стратегий для каждого детектора

8. **Human-in-the-loop**: пользователь выбирает из 3 вариантов перефразирования каждого предложения

## 8. Честный вывод

TextHumanize v0.25.0+ — это:
- ✅ **Хорошая локальная библиотека** для снижения "AI-звучания" текста
- ✅ **0 зависимостей**, работает офлайн, приватность 100%
- ✅ **Быстрая** (~1.5 сек на текст vs 5-15 сек у конкурентов)
- ❌ **Не может bypass** современные нейронные AI-детекторы
- ❌ **Уступает конкурентам** по качеству bypass в 10-20 раз
- ❌ **Качество текста** страдает от агрессивных трансформаций

**Для реального bypass нужен LLM-rewriting.** Все коммерческие конкуренты это поняли и используют облачные LLM. Наша ниша — приватность + бесплатность + локальность. Для bypass нужно добавить Ollama/LLM-backend (уже есть инфраструктура, нужна реализация).
